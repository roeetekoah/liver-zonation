Differential Expression & the False Discovery Rate

From a single p-value to honest large-scale discovery: p-values, Bonferroni, Benjamini–Hochberg, and q-values.
A recap of Lecture 5 — p-values, multiple hypothesis testing, and FDR.
Following the lecture's framing: from a single anomalous coverage region to testing the whole genome.

“Throw enough nets into the sea and you will catch plastic bottles, not just fish.”

1. The setting: from one region to many tests

Lecture 5 starts where the previous lectures left off. We had modelled genome coverage with a Poisson distribution: the probability of coverage starting at position i over length k is Poisson in the requested depth and length. That Poisson model is our null hypothesis H0 — it describes a typical region, where coverage is random and independent of position.

The biological question is: are there special regions? Regions that “attract” or “repel” coverage, that carry more errors, that simply do not look like the null predicts? Given some observed coverage at some sequencing depth, we want to ask: is this anomalous or not, and how do we quantify it?

Recap from CBIO There we discussed separating two simple hypotheses — likelihood-ratio tests, Neyman–Pearson. Today is different: we have only H0, and no alternative. We are not asking “which of two hypotheses is more likely?” but “how unlikely is what we saw, assuming H0?” That single question is what the p-value answers.

What “differential expression” is, in this language

In a single-cell / bulk genomics context, the very same machinery powers differential gene expression (DE). For each gene we ask whether its expression differs between two conditions or groups (e.g. treated vs. control, cell-type A vs. B). The per-gene null and alternative are:

H0,g: gene g is not differentially expressed (same distribution in both groups)
H1,g: gene g is differentially expressed

We compute a per-gene test statistic, turn it into a p-value, and — crucially — we do this for every gene at once (~20,000 of them). That is exactly the “many regions in the genome” problem the lecture builds toward, and it is where naive p-value thresholds fall apart.

2. The p-value

Start with the abstract definition the lecture uses. We have a random variable X and a null distribution P0 on X. In our case X is the coverage and P0 is the Poisson model. We observe a single value x and ask how surprising it is under P0. The (right-tail) p-value is:

pV(X) = P0( X ≥ x )

Intuitively: the chance of getting the observed value or anything more extreme. The lecture is careful here — we use a tail probability, not a point probability. A single point can have tiny (or even zero, in the continuous case) probability while the whole region is perfectly ordinary. The tail is what measures “extremeness.”

When is a result surprising? When the p-value is small. We fix a significance threshold α, and declare: if pV < α, the result is “significant” — meaning significantly abnormal, i.e. x is an outlier. How unlikely? “α-unlikely.”

Worked example from the lecture

How surprising is coverage of 52 on a depth of D = 10? We compute the tail probability under the Poisson null. The complement / CDF form is easier to compute, and the answer is: getting coverage 52 when the expected depth is 10 is extremely unlikely — a tiny p-value, a clear outlier.

What a p-value is NOT A p-value is P(data this extreme | H0). It is not P(H0 is true | data). The null is not a random hypothesis — it is the fixed baseline (“no effect”, “nothing special”). To go from P(data | H0) to P(H0 | data) you would need a prior on H0 and Bayes' rule. The p-value is strictly one-directional.

Two things to always keep in mind

  • pV is a function of H0. Always ask: which null am I testing against? Write it as pV(X; H0).
  • pV always involves a tail (cumulative) probability. For Poisson or geometric this is trivial; for many distributions the tail is exactly the hard part, needing approximations.
A subtle but central fact If the test statistic is continuous, then under H0 the p-value is uniform on [0,1]: P0( pV(X) ≤ ρ ) = ρ. Reason: pV ≤ ρ is exactly the event “my statistic landed in the top ρ-fraction of the null tail”, and under the null the statistic is a draw from that distribution, so it lands there exactly a ρ-fraction of the time. This calibration is the foundation everything below rests on. (For discrete statistics you get ≤ ρ instead of equality.)

3. The multiple-testing problem

Now we change perspective: instead of one region with high coverage, we scan many regions across the genome and ask which look surprising. We get a whole vector of p-values, one per region:

pV1, pV2, …, pVm  —  m separate null hypotheses (same distribution, different regions)

What do we expect? Suppose we use α = 0.05. The chance any single region is flagged anomalous by chance is 1 in 20. But now we test, say, 20,000 hypotheses at once. The number flagged is a sum of indicators “pVi < α”, and by linearity of expectation:

E[ # flagged ] = m · α = 20,000 · 0.05 = 1,000
The core danger Even if nothing interesting is happening — every null is true — running 20,000 tests at α=0.05 produces about 1,000 false positives by pure chance. Roll a die enough times and even “1-in-20” events happen constantly. Real signal gets buried in noise. This is why so many published “extraordinary” findings are not extraordinary at all: with enough tests, something striking is guaranteed to appear (the “look-elsewhere” effect).

The central question of the rest of the lecture: how do we correct the framework so we don't get α·m spurious “discoveries” when the null is true? Since the false-positive count is linear in α and in the number of tests, every correction revolves around shrinking or reinterpreting α.

Quantity to controlNamePlain meaning
P( at least one false positive )FWER (family-wise error rate)“I refuse to report even one wrong result.”
E[ false positives / total called ]FDR (false discovery rate)“A small fraction of my reported hits may be wrong.”

4. Bonferroni: control the family-wise error

The Bonferroni correction takes a corrected threshold:

αBonf = α / m

Why does this work at scale? We want to bound the probability that even one of the m tests drops below the threshold under H0. This is a question about a union of events, so we apply the union bound:

P( ∃ i : pVi ≤ α/m ) ≤ ∑i P( pVi ≤ α/m ) = m · (α/m) = α
What Bonferroni buys The correction restores α for the whole system: the probability of getting even one spurious hit at level 0.05 across all m tests is back to 0.05. If you want a family-wise error of α over m tests, test each at α/m. The more questions you ask, the smaller the per-test threshold.
The problem with Bonferroni It asks too strict a question. It controls the chance that anyone at all crosses the threshold — it tries to guarantee almost no false positive ever. In real genomics, with 20,000 genes, α/m becomes so tiny (e.g. 0.05/20000 = 0.0000025) that you lose almost all real biological signal. We usually do not need “no false positives ever”; we need to control the expected fraction of false hits among those we report.

5. FDR and the Benjamini–Hochberg procedure

Why is FDR a better fit for multiple testing? Because the population of tests usually hides two groups: a large majority that genuinely follow H0, and a small hidden group of truly significant effects. Bonferroni, in its zeal, misses the small real group. FDR instead controls the rate of false discoveries rather than forbidding them.

The fishing-net intuition Throw a net into the sea. You may catch plastic bottles instead of fish. We don't demand that every cast contains only fish (impossible). We demand that most of what we haul up is fish. FDR controls the share of junk in the catch, not the existence of junk.

Defining FDR

Among the regions/genes we call positive, the FDR is the expected fraction that are actually false positives:

FDR = E[ V / max(R, 1) ]    V = #false positives,  R = #total called

Controlling FDR at 0.05 says: of the things I report, on average at most 5% are noise. Unlike Bonferroni (“1-in-20 chance of anything spurious”), FDR says “take the whole pile of hits, and at most 1-in-20 of them is wrong.”

Deriving the BH threshold

Sort the p-values from most to least surprising and consider declaring the first k of them discoveries:

Y1 ≤ Y2 ≤ … ≤ Ym  —  cut after k, using threshold p = Yk

Under the null, P(p-value ≤ p) = p, so among m tests the expected number of false discoveries at threshold p is at most m·p. With k discoveries total, the expected false fraction is bounded by m·p / k. Demanding this be ≤ α and substituting p = Yk:

m · Yk / k ≤ α  ⇒   Yk ≤ (k / m) · α
Benjamini-Hochberg procedure  (control FDR at level q)
------------------------------------------------------
1. Sort all m p-values in increasing order:
       Y(1) <= Y(2) <= ... <= Y(m)

2. For each rank k, compare to the BH line:
       Y(k)  <=  (k / m) * q   ?

3. Find the LARGEST k for which this holds.   <-- the cutoff

4. Declare ALL tests with rank <= k significant
   (even any whose own p-value is above its own line).
Why the moving threshold? Bonferroni uses one tiny fixed threshold α/m for every test. BH uses (k/m)·α, which grows with k. At k=1 it equals Bonferroni; for larger k it relaxes. The logic: if you are already making many discoveries, tolerating a few false ones is fine as long as their fraction stays below α.

6. Worked BH mini-example

Take m = 5 tests with ordered p-values, and control FDR at α = 0.05. The BH line at rank k is (k/5)·0.05.

The largest k satisfying Y(k) ≤ (k/m)·α is k = 3. So we declare ranks 1, 2, 3 as discoveries. Note this is a step-up rule: we find the rightmost crossing and accept everything to its left, even if an intermediate point sat just on or below its own line.

Reading the last column The final column, m·Y(k)/k, is essentially the q-value of each test: the smallest FDR level at which that test would be called significant. Result 1 needs only FDR 1%; result 3 needs FDR 5%; result 4 would need FDR 25%. (A small monotonicity fix makes q-values formally non-decreasing, but this is the idea.)

7. q-values and how significance is reported

Raw p-values are calibrated for one hypothesis. But in large-scale testing the question we actually care about is different: if I put this result on my discovery list, how reliable is that list? A p-value of 0.01 across 20,000 genes is not impressive — under pure noise you expect 20,000·0.01 = 200 such hits. So we report a q-value instead: a p-value recalibrated for the whole search.

q-value, defined The q-value of a test is the smallest FDR level at which that test would be called significant by the BH procedure. Equivalently: among all discoveries at least this significant, the expected fraction that are false. The p-value is to a single test what the q-value is to FDR.

BH and q-values solve the same problem from opposite ends: BH fixes α first and returns the discovery set; the q-value attaches to each test the minimal α that would include it. Reporting q-values lets a reader pick their own tolerance — accept q<0.05 for a strict list, q<0.10 for a broader one — without rerunning the procedure.

Refinements the lecture flags

The bound “expected false discoveries ≤ m·p” pessimistically assumes all hypotheses are null. If only m0 are truly null, the real expectation is m0·p < m·p. Adaptive methods estimate the null proportion π0 = m0/m and use π0·m·p instead, recovering more power (Storey's q-value). The plain BH bound is tightest when m is large and most tests are null.

Effect size matters alongside p/q A tiny q-value says a difference is real, not that it is large. In DE analysis, always report the effect size (e.g. log fold-change) next to the p/q-value. A gene can be highly significant yet biologically trivial; volcano plots exist precisely to show both axes at once.
Assumptions & test choice (DE / single-cell) BH controls FDR under independence or positive dependence of the tests. Heavily / negatively correlated tests need Benjamini–Yekutieli (more conservative). The p-values themselves must be calibrated: a wrong null (e.g. Poisson when counts are over-dispersed) distorts interpretation — negative-binomial models or empirical nulls fit count data better. Choose the per-gene test to match the data; FDR correction is only as honest as the p-values feeding it.
Core ideas to remember (1) p-value = P(data this extreme | H0), not P(H0 true | data).  (2) ~20,000 tests at α=0.05 → ~1,000 false positives by chance — correction is mandatory.  (3) Bonferroni (α/m) controls FWER — safe but loses real signal at scale.  (4) BH controls FDR via the largest k with Y(k) ≤ (k/m)·α.  (5) q-value = smallest FDR at which a test is called; report q<0.05 with an effect size.
